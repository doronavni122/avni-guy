# OWNER-REPORTS — working reports (non-authoritative)

This directory holds **session and program workfiles** only: spike notes, audits, gap lists, and operator checklists.

## Not product law

- Do **not** treat files here as SSOT for product, API contracts, or security baselines.
- Prefer **code**, `END-GOAL-PROJECT.md`, `PROJECT_USER_JOURNEY_STEPS.md`, and formal ADRs in `ADR/`.

## Allowed use

- Historical context, investigation pointers, temporary tracking until work lands in code or ADRs.

## Naming (suggested)

- `NNN_<topic>_<YYYY-MM-DD>.md` or `*_spike_audit.md`, `*_gap_inventory.txt` — dense, machine-parseable.

Add new reports here; do not commit secrets or production credentials.
